# FPGA 音乐播放器实现说明

## 1. 总体结构

工程采用模块化结构，`music_top.v` 负责连接各功能模块。整体数据流如下：

```text
按键
  -> button_debounce
  -> ui_controller / player_controller
  -> song_rom
  -> pitch_processor
  -> beat_controller
  -> tone_generator
  -> beep

当前音符、时间、参数
  -> led_panel_controller / sevenseg_ui_formatter
  -> board_led_mapper / sevenseg_scan_controller
  -> LED 和数码管
```

主要模块如下：

| 文件 | 作用 |
| --- | --- |
| `music_top.v` | 顶层连接、播放计时、模块实例化 |
| `button_debounce.v` | 按键消抖，并产生单周期按下脉冲 |
| `ui_controller.v` | 乐曲、移调、BPM、音量四个参数的编辑状态机 |
| `player_controller.v` | 播放、暂停、停止和音符索引控制 |
| `song_rom.v` | 内置乐曲 ROM 和乐曲元数据 |
| `pitch_processor.v` | 根据音符编码和移调值计算发声音高与显示音名 |
| `beat_controller.v` | 根据 BPM 和音符时值产生节拍、十六分音符和音符结束脉冲 |
| `tone_generator.v` | 把半音音高转换为蜂鸣器方波 |
| `led_panel_controller.v` | 生成 4 行 LED 的逻辑显示内容 |
| `board_led_mapper.v` | 将逻辑 LED 行列映射到原理图 LED 编号 |
| `sevenseg_ui_formatter.v` | 将参数和播放时间格式化为 8 个数码管 glyph |
| `sevenseg_scan_controller.v` | 动态扫描 8 位数码管 |

## 2. 歌曲和音符编码

`song_rom.v` 中每个音符使用 16 位编码：

| 位段 | 字段 | 含义 |
| --- | --- | --- |
| `[15]` | `rest` | 1 表示休止符，0 表示普通音符 |
| `[14:12]` | `note_name` | C、D、E、F、G、A、B |
| `[11:10]` | `accidental` | 降号、自然、升号 |
| `[9:6]` | `octave` | 八度，范围 0 到 8 |
| `[5:0]` | `duration_16th` | 以十六分音符为单位的时值 |

代码中通过函数写音符，而不是手动拼二进制：

```verilog
note(N_C, ACC_NATURAL, 4'd4, DUR_QUARTER)
rest(DUR_QUARTER)
```

这种写法比直接写 `16'b...` 更适合维护，因为音高、升降号、八度和时值都能直接读出来，也方便后续添加歌曲。

ROM 除了输出当前 `note_word`，还输出每首歌的元数据：

| 字段 | 作用 |
| --- | --- |
| `song_length` | 当前乐曲音符数量 |
| `total_duration_16th` | 当前乐曲总时值，用于播放进度条 |
| `default_bpm` | 当前乐曲默认 BPM |
| `key_tonic`、`key_accidental`、`key_mode` | 调性元数据，当前主要作为保留信息 |
| `beats_per_bar` | 每小节拍数，当前用于 LED 拍位显示 |
| `first_beat_in_bar` | 第一颗音从小节内第几拍开始，0 表示第一拍 |

当前实现的移调算法不依赖调号元数据，而是根据原始音符拼写和移调音程计算显示拼写。调号元数据保留下来，是为了以后实现更完整的调性显示或按调号自动记谱。

## 3. 移调和音高显示

`pitch_processor.v` 同时计算两个结果：

1. `semitone_pitch`：用于蜂鸣器实际发声的半音音高，类似 MIDI 音高编号。
2. `display_note_name/display_accidental/display_octave`：用于 LED 显示的音名、升降号和八度。

计算步骤：

1. 根据原始音名、升降号和八度得到原始半音音高。
2. 加上 `transpose_semitones` 得到目标发声音高。
3. 根据移调半音数推算音程度数变化，例如 +1/+2 半音都移动到下一个字母音名，+3/+4 半音移动两个字母音名。
4. 用目标发声音高减去目标自然音的音高，得到升号、降号或自然号。

例子：

| 原音 | 移调 | 发声音高 | 显示 |
| --- | --- | --- | --- |
| C4 | +1 | 61 | Db4 |
| C4 | +2 | 62 | D4 |
| C4 | +6 | 66 | F#4 |
| C4 | -1 | 59 | B3 |

这样可以区分听起来相同但记谱不同的音，例如 C# 和 Db。当前算法是“按音程关系推导拼写”，不是只按 12 平均律半音编号查表。

## 4. BPM 和节拍时序

`beat_controller.v` 使用 BPM 和音符时值控制播放长度。由于 BPM 表示每分钟四分音符数量，而一个四分音符等于 4 个十六分音符，所以一个十六分音符的时钟周期数为：

```text
ticks_per_16th = 15 * CLK_FREQ_HZ / BPM
```

模块输出：

| 信号 | 含义 |
| --- | --- |
| `sixteenth_pulse` | 每经过一个十六分音符产生一个脉冲 |
| `beat_pulse` | 每 4 个十六分音符产生一个四分音符节拍脉冲 |
| `note_done` | 当前音符时值结束 |
| `tone_enable` | 当前是否允许蜂鸣器发声 |

为了让相邻音符边界更清晰，模块在音符末尾加入一个很短的静音间隙，默认由 `NOTE_GAP_MS` 控制。

## 5. 播放和参数状态机

`player_controller.v` 管理播放状态：

| 状态 | 行为 |
| --- | --- |
| stopped | 停止在第 0 个音符 |
| playing | 根据 `note_done` 推进音符 |
| paused | 保持当前音符索引，蜂鸣器关闭 |

`ui_controller.v` 管理三个可编辑参数：

| 编辑模式 | 范围 | 显示 |
| --- | --- | --- |
| 乐曲 | 0 到 1 | `S001`、`S002` |
| 移调 | -12 到 +12 | `t+00` 到 `t-12` |
| BPM | 30 到 250 | `b030` 到 `b250` |
| 音量 | 0 到 7 | `U000` 到 `U007` |

`key_next` 让编辑模式按“乐曲 -> 移调 -> BPM -> 音量 -> 乐曲”循环。5 秒无操作会自动回到乐曲显示模式。

音量值由 `ui_controller.v` 输出到 `tone_generator.v`。当前音量曲线使用蜂鸣器输出占空比控制：

| 音量 | 占空比 |
| --- | --- |
| 0 | 静音 |
| 1 | 约 0.78% |
| 2 | 约 1.56% |
| 3 | 约 3.12% |
| 4 | 约 6.25% |
| 5 | 约 12.5% |
| 6 | 约 25% |
| 7 | 约 50% |

## 6. LED 显示实现

`led_panel_controller.v` 生成四行逻辑 LED：

| 行 | 逻辑 |
| --- | --- |
| row0 | 当前音名 one-hot，C 到 B 对应 bit0 到 bit6 |
| row1 | 当前八度 one-hot，1 到 8 对应 bit0 到 bit7 |
| row2 | bit0 降号，bit1 升号，bit4 到 bit7 显示当前小节内拍位 |
| row3 | 播放进度条 |

`board_led_mapper.v` 负责把逻辑行列映射到原理图中的 `led[31:0]`。这样做的好处是显示逻辑可以按照片直观编写，而 XDC 仍按原理图 LED 编号约束。

拍位显示使用右对齐规则：

| 拍号 | 可用灯位 | 第一拍位置 |
| --- | --- | --- |
| 2/4 | row2[6]、row2[7] | row2[6] |
| 3/4 | row2[5]、row2[6]、row2[7] | row2[5] |
| 4/4 | row2[4]、row2[5]、row2[6]、row2[7] | row2[4] |

任意时刻只点亮当前拍对应的一个灯，并保持到下一次 `beat_pulse`。

## 7. 数码管实现

数码管分两层：

1. `sevenseg_ui_formatter.v`：把参数和时间转换成 8 个 5-bit glyph。
2. `sevenseg_scan_controller.v`：把 glyph 解码成 A/B/C/D/E/F/G/DP 段码，并动态扫描 8 个位选。

逻辑位编号从右到左：

| 逻辑位 | 位置 | 内容 |
| --- | --- | --- |
| digit0 | 最右 | 秒个位 |
| digit1 | 右二 | 秒十位 |
| digit2 | 右三 | 分钟个位，带小数点 |
| digit3 | 右四 | 分钟十位 |
| digit4 到 digit7 | 左四位 | 参数状态，包括 `S001`、`t+00`、`b120`、`U007` |

段线顺序为 A、B、C、D、E、F、G、DP。当前参数配置为段线高有效、位选低有效：

```verilog
.SEG_ACTIVE_HIGH(1),
.CS_ACTIVE_LOW(1)
```

如果上板后发现数码管极性相反，可以优先调整这两个参数。

## 9. 后续维护建议

添加新歌曲时：

1. 在 `song_rom.v` 中增加新的 `song_select` 分支。
2. 设置 `song_length`、`total_duration_16th`、`default_bpm` 和调性元数据。
3. 用 `note(...)` 和 `rest(...)` 编写音符序列。
4. 增大 `ui_controller` 实例中的 `SONG_COUNT`，并扩展 `selected_song` 位宽。
5. 更新测试平台和说明文档。

调整硬件显示时：

1. 如果是 LED 物理位置不对，优先修改 `board_led_mapper.v`。
2. 如果是 LED 原理图引脚不对，修改 `music.xdc` 中的 `led[n]`。
3. 如果是数码管位顺序不对，优先检查 `seg_cs[n]` 映射。
4. 如果是数码管段顺序不对，优先检查 `seg[n]` 的 A/B/C/D/E/F/G/DP 映射。

音量控制已经合入 `ui_controller.v`，旧的独立 `volume_controller` 模块已从顶层文件中移除。
