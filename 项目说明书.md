# FPGA 音乐播放器项目说明书

## 1. 项目概述

本项目是在 Vivado 2018.1 工程 `lab_music` 上实现的 FPGA 简易音乐播放器。系统可以播放内置乐曲，通过蜂鸣器输出方波音频，并使用 32 个独立 LED 和 8 位数码管显示当前音符、八度、升降号、节拍、播放进度、播放时间、剩余时间、小节拍位和当前可编辑参数。系统还支持播放模式选择、参数按键长按连发、板载自检模式和乐谱转换工具。

工程入口为：

```text
lab_music/lab_music.xpr
```

顶层模块为：

```text
lab_music/lab_music.srcs/sources_1/new/music_top.v
```

当前 FPGA 型号和工程器件设置为 `xc7k325tffg900-2`。约束文件为：

```text
lab_music/lab_music.srcs/constrs_1/new/music.xdc
```

## 2. 硬件资源分配

### 2.1 时钟、按键和蜂鸣器

| 功能 | 板上资源 | FPGA 引脚 | 有效电平 |
| --- | --- | --- | --- |
| 系统时钟 | CLK | AD12 | 200 MHz |
| 复位 | KEY8 | D11 | 低有效 |
| 播放/暂停 | KEY7 | C11 | 低有效 |
| 停止 | KEY6 | E14 | 低有效 |
| 参数选择 | KEY5 | B13 | 低有效 |
| 参数减小 | KEY4 | A13 | 低有效 |
| 参数增大 | KEY3 | D14 | 低有效 |
| 播放显示模式切换 | KEY2 | C14 | 低有效 |
| 板载自检模式 | KEY1 | E15 | 低有效 |
| 蜂鸣器 | BEEP | W19 | 方波输出 |

代码中仍保留了历史端口名 `key_volume_down` 和 `key_volume_up`，但当前含义已经改为“参数减小”和“参数增大”。KEY3/KEY4 支持长按自动连发，按住一段时间后会自动重复触发，继续长按会进入更快的连发速度。

### 2.2 32 个独立 LED

LED 按板子照片中的物理位置抽象为四行，每行 8 个灯，从左到右编号为 bit0 到 bit7。实际原理图 LED 编号由 `board_led_mapper.v` 负责转换。

| 逻辑行 | 显示内容 |
| --- | --- |
| 第 0 行 | 当前音名，前 7 个灯依次表示 C、D、E、F、G、A、B |
| 第 1 行 | 当前八度，0 八度全灭，1 到 8 八度分别点亮一个灯 |
| 第 2 行 | bit0 表示降号，bit1 表示升号，右侧 4 个灯显示小节内当前拍 |
| 第 3 行 | 当前乐曲播放进度条 |

### 2.3 8 位数码管

数码管由 4 个两位数码管模块组成。代码中把它抽象为 8 个逻辑位，从右到左编号为 `digit0` 到 `digit7`：

| 数码管区域 | 显示内容 |
| --- | --- |
| 左边四位 | 当前正在编辑或默认显示的参数 |
| 右边四位 | KEY2 选择的播放显示内容 |

左四位显示格式：

| 参数 | 显示例子 | 说明 |
| --- | --- | --- |
| 乐曲编号 | `S001`、`S002`、`S003` | 显示为 1 开始编号，内部仍是 0 开始 |
| 移调 | `t+00`、`t-12` | 范围为 -12 到 +12 半音 |
| BPM | `b120` | 范围为 30 到 250 |
| 音量 | `U007` | 范围为 0 到 7，0 为静音 |
| 播放模式 | `P-St`、`P-1L`、`P-AL` | 分别表示播完停止、单曲循环、全部循环 |

右四位显示模式由 KEY2 循环切换：

| 模式 | 显示格式 | 说明 |
| --- | --- | --- |
| 已播放时间 | `MM.SS` | 当前乐曲已经播放的分钟和秒 |
| 剩余时间 | `MM.SS` | 按当前 BPM 和乐曲总时值估算的剩余时间 |
| 小节拍位 | `BBB.P` | 前三位为小节号，最后一位为小节内第几拍，例如 `123.1` |

暂停时，右侧四位显示会闪烁；左侧参数显示不闪烁。

## 3. 操作说明

1. 按 KEY8 复位系统。
2. 按 KEY7 开始播放；播放中再次按 KEY7 暂停；暂停时再按 KEY7 继续播放。
3. 按 KEY6 停止播放，并回到当前乐曲的第一个音。
4. 按 KEY5 在“乐曲编号、移调、BPM、音量、播放模式”五个参数之间循环选择。
5. 按 KEY4 减小当前参数，按 KEY3 增大当前参数；长按 KEY3/KEY4 会自动连发。
6. 按 KEY2 在右侧显示的“已播放时间、剩余时间、小节拍位”之间循环切换。
7. 长按 KEY1 进入或退出板载自检模式。自检时 LED 单点扫描，数码管显示 `01234567`，蜂鸣器输出固定诊断音。
8. 如果 5 秒没有进行参数操作，显示会自动回到乐曲编号模式。

切换乐曲时，播放器会自动停止并把音符索引清零。
播放模式为 `P-St` 时，乐曲结束后停止；`P-1L` 时当前乐曲循环；`P-AL` 时播放完当前乐曲后自动切换到下一首，最后一首结束后回到第一首。

## 4. 当前内置乐曲

| 显示编号 | 内部编号 | 乐曲 |
| --- | --- | --- |
| `S001` | 0 | Twinkle Twinkle Little Star |
| `S002` | 1 | Old Memory MIDI melody |
| `S003` | 2 | Haruhikage MIDI melody |

第一首曲子的默认 BPM 是 120，调性元数据为 C 大调。第二首由 `old memory.mid` 转换得到，默认 BPM 为 85，拍号为 4/4，调性元数据为 D 大调。第三首由 MIDI 文件转换得到，默认 BPM 为 200，拍号为 3/4，调性元数据为 B 大调。

## 5. 显示现象参考

系统复位后，默认乐曲为 `S001`，移调为 `t+00`，BPM 为 `b120`。当前音符从第一首歌第一个音开始，因此 LED 音名行会显示 C，八度行会显示第 4 八度。
默认音量为 `U007`。音量为 0 时静音，1 到 7 档使用逐步增大的蜂鸣器高电平占空比。

播放时：

- 蜂鸣器输出当前音高对应的方波。
- 每个音根据自身时值持续播放。
- 第三行右侧 4 个 LED 按拍号右对齐显示当前小节内拍位，并保持到下一拍。
- 最后一行 LED 根据整首歌的十六分音符进度逐步点亮。
- 右侧数码管根据 KEY2 选择显示经过时间、剩余时间或小节拍位。

## 7. Vivado 常见问题

### 7.1 别人拉取后提示 `module 'key_repeat' not found`

`key_repeat.v`、`long_press_toggle.v` 和 `self_test_controller.v` 是新增模块，已经被加入 Git 和 `lab_music.xpr`。如果别人拉取后综合仍提示找不到 `key_repeat`，通常是 Vivado 打开的工程缓存没有刷新，或者打开了旧的 `.xpr`。处理方法如下：

1. 确认已经执行 `git pull`，并且文件 `lab_music/lab_music.srcs/sources_1/new/key_repeat.v` 存在。
2. 关闭 Vivado 后重新打开 `lab_music/lab_music.xpr`。
3. 如果仍报错，在 Vivado Tcl Console 中执行：

```tcl
add_files -norecurse lab_music.srcs/sources_1/new/key_repeat.v
add_files -norecurse lab_music.srcs/sources_1/new/long_press_toggle.v
add_files -norecurse lab_music.srcs/sources_1/new/self_test_controller.v
update_compile_order -fileset sources_1
reset_run synth_1
launch_runs synth_1
```

4. 也可以删除 `.Xil`、`lab_music.cache`、`lab_music.runs`、`lab_music.sim` 等生成目录后重新打开工程。这些目录是 Vivado 生成物，不需要提交。

### 7.2 `open_hw_target` 提示没有检测到设备

类似下面的报错不是综合错误，而是别人的电脑没有连接到同一个下载器或板卡：

```text
No devices detected on target localhost:3121/xilinx_tcf/Digilent/...
```

换机器后需要在 Hardware Manager 中重新连接本机的硬件目标，或在 Tcl Console 中执行：

```tcl
disconnect_hw_server
connect_hw_server
open_hw_target
```

如果只做综合、实现或仿真，可以忽略硬件目标报错。

## 8. 上板注意事项

由于当前没有实物板进行最终确认，以下硬件对应关系需要上板后重点观察：

1. 32 个独立 LED 的物理行列方向是否与照片一致。
2. 数码管是否为段线高有效、位选低有效。
3. `LED3/LED4` 两个双位数码管的左右位选是否需要互换。
4. 如果数码管某些段显示错位，应优先检查 XDC 中 `seg[n]` 的 A/B/C/D/E/F/G/DP 顺序。
5. 如果数码管只显示一位或完全不亮，应优先检查 `seg_cs[n]` 的有效电平和左右位选顺序。

上述问题通常只需要修改 `music.xdc` 或 `sevenseg_scan_controller.v` 的极性参数，不需要改动歌曲、移调或播放器核心逻辑。
